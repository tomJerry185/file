## 1. 数据分级表

> MUST：所有业务数据必须按以下分级标准分类管理。

| 级别 | 标识 | 定义 | 示例 | 泄露影响 |
|------|------|------|------|----------|
| L4 - 机密 | Confidential | PII 敏感信息 | 身份证号、银行卡号、密码哈希 | 法律风险、重大财产损失 |
| L3 - 敏感 | Sensitive | 可关联个人的信息 | 手机号、邮箱、地址、真实姓名 | 骚扰、诈骗风险 |
| L2 - 内部 | Internal | 业务运营数据 | 订单金额、库存数量、用户ID | 商业竞争损失 |
| L1 - 公开 | Public | 可公开信息 | 商品名称、商品描述、评价内容 | 低影响 |

### 1.1 字段级分级清单

| 数据字段 | 级别 | 存储要求 | 日志要求 | API 要求 |
|----------|------|----------|----------|----------|
| 密码 | L4 | bcrypt/argon2 哈希 | 禁止记录 | 禁止返回 |
| 身份证号 | L4 | AES-256-GCM 加密 | 禁止记录 | 仅返回后4位 |
| 银行卡号 | L4 | AES-256-GCM 加密 | 禁止记录 | 仅返回后4位 |
| 手机号 | L3 | 可逆加密（AES-128） | 脱敏：138\*\*\*\*1234 | 脱敏返回 |
| 邮箱 | L3 | 明文或可逆加密 | 脱敏：u\*\*\*@example.com | 脱敏返回 |
| 收货地址 | L3 | 可逆加密 | 脱敏：省市区+\*\*\*\* | 脱敏返回 |
| 用户姓名 | L3 | 明文 | 脱敏：张\* | 普通用户脱敏 |
| 订单金额 | L2 | 明文 | 明文 | 管理员可见 |
| 订单号 | L2 | 明文 | 明文 | 明文 |
| 商品信息 | L1 | 明文 | 明文 | 明文 |

## 2. 加密存储规范

> MUST：L4 数据必须加密存储，L3 数据建议加密存储。

### 2.1 加密算法要求

| 用途 | 算法 | 密钥长度 | 规则编号 |
|------|------|----------|----------|
| 密码哈希 | Argon2id（优先）或 bcrypt | bcrypt cost=12 | SEC-CRYPT-001 |
| 对称加密 | AES-256-GCM | 256 bit | SEC-CRYPT-002 |
| 非对称加密 | RSA-2048（最低）或 RSA-4096 | 2048/4096 bit | SEC-CRYPT-003 |
| 令牌签名 | HMAC-SHA256 | 256 bit | SEC-CRYPT-004 |

### 2.2 密码存储规范

MUST：密码永远以哈希形式存储，禁止明文或可逆加密。

```text
// 推荐算法
HashAlgorithm: Argon2id
  - Memory: 64 MB
  - Iterations: 3
  - Parallelism: 4
  - Salt: 16 bytes random

// 备选算法
HashAlgorithm: bcrypt
  - Cost factor: 12
  - Auto-generated salt
```

MUST：禁止使用 MD5、SHA1、SHA256 单独作为密码哈希。

### 2.3 敏感字段加密

```java
// AES-256-GCM 加密示例
@Configuration
public class EncryptionConfig {
    @Value("${encryption.master-key}")
    private String masterKey; // 从 KMS 或环境变量获取

    public String encrypt(String plaintext) {
        // 1. 生成随机 IV (12 bytes)
        // 2. AES-256-GCM 加密
        // 3. 返回 Base64(IV + ciphertext + authTag)
    }
}
```

### 2.4 密钥管理

MUST：加密密钥禁止硬编码在代码或配置文件中。

| 密钥类型 | 存储方式 | 轮换周期 |
|----------|----------|----------|
| 数据加密密钥（DEK） | KMS（阿里云 KMS / AWS KMS） | 每年 |
| 密钥加密密钥（KEK） | KMS 硬件安全模块 | 每 2 年 |
| JWT 签名密钥 | 环境变量 / KMS | 每 90 天 |
| API 密钥 | 密钥管理服务 | 每年 |

## 3. 传输加密规范

> MUST：所有外部通信必须使用 TLS 1.2+。

### 3.1 传输加密要求

| 场景 | 要求 | 规则编号 |
|------|------|----------|
| 外部 HTTPS | TLS 1.2+，禁用 TLS 1.0/1.1 | SEC-TLS-001 |
| 内部微服务通信 | TLS 1.2（推荐 mTLS） | SEC-TLS-002 |
| 数据库连接 | SSL/TLS 或内网隔离 | SEC-TLS-003 |
| Redis 连接 | TLS 或内网隔离 | SEC-TLS-004 |
| MQ 通信 | ACL + 内网隔离 | SEC-TLS-005 |

### 3.2 证书管理

MUST：TLS 证书到期前 30 天自动提醒。

```text
提醒时间线：
- 到期前 30 天：邮件通知 DevOps
- 到期前 14 天：钉钉/飞书群通知
- 到期前 7 天：升级为 P1 告警
- 到期：自动续期（ACME/Let's Encrypt）或紧急处理
```

## 4. 日志脱敏规则

> MUST：L3/L4 数据在日志中必须脱敏。

### 4.1 脱敏规则表

| 数据类型 | 原始格式 | 脱敏后格式 | 规则编号 |
|----------|----------|------------|----------|
| 手机号 | 13812341234 | 138\*\*\*\*1234 | SEC-MASK-001 |
| 身份证 | 310101199001011234 | 310\*\*\*\*\*\*\*\*\*\*\*1234 | SEC-MASK-002 |
| 银行卡 | 6222021234567890123 | 6222 \*\*\* \*\*\*\* 0123 | SEC-MASK-003 |
| 邮箱 | user@example.com | u\*\*\*@example.com | SEC-MASK-004 |
| 姓名 | 张三丰 | 张\*\* | SEC-MASK-005 |
| 地址 | 北京市朝阳区XX路100号 | 北京市朝阳区\*\*\*\* | SEC-MASK-006 |
| 密码 | 任意 | [PROTECTED] | SEC-MASK-007 |

### 4.2 实现方式

MUST：使用 Logback Masking 或 Jackson Serializer 自动脱敏。

```java
// 使用注解标记敏感字段
public class UserDTO {
    @Sensitive(strategy = SensitiveStrategy.PHONE)
    private String phone;

    @Sensitive(strategy = SensitiveStrategy.ID_CARD)
    private String idCard;

    @Sensitive(strategy = SensitiveStrategy.EMAIL)
    private String email;
}
```

MUST：禁止在日志中输出密码、Token、密钥等 L4 数据。

## 5. API 响应脱敏

> MUST：API 响应根据调用方角色返回不同粒度的数据。

### 5.1 角色数据可见性矩阵

| 数据字段 | 普通用户 | 管理员 | 客服 | 系统间调用 |
|----------|----------|--------|------|------------|
| 密码 | 禁止 | 禁止 | 禁止 | 禁止 |
| 身份证号 | 禁止 | 后4位 | 后4位 | 完整（需审计） |
| 手机号 | 脱敏 | 完整 | 脱敏 | 完整（需审计） |
| 银行卡号 | 后4位 | 后4位 | 后4位 | 完整（需审计） |
| 收货地址 | 脱敏 | 完整 | 省市区 | 完整 |
| 订单金额 | 本人可见 | 完整 | 本人可见 | 完整 |
| 用户姓名 | 本人可见 | 完整 | 脱敏 | 完整 |

### 5.2 实现方式

MUST：使用 Jackson `@JsonView` 或自定义 Serializer 控制响应字段。

```java
// 使用 @JsonView 控制不同角色的可见字段
public class UserVO {
    @JsonView(Views.Public.class)
    private Long id;

    @JsonView(Views.Public.class)
    private String nickname;

    @JsonView(Views.Owner.class)
    private String phone; // 仅本人可见

    @JsonView(Views.Admin.class)
    private String realName; // 仅管理员可见
}
```

## 6. 数据保留与清理策略

> MUST：各类数据有明确的保留期限和清理机制。

| 数据类型 | 保留期限 | 清理方式 | 规则编号 |
|----------|----------|----------|----------|
| 操作日志 | 30 天 | 自动归档后删除 | SEC-RET-001 |
| 登录日志 | 90 天 | 自动删除 | SEC-RET-002 |
| 审计日志 | 1 年 | 归档至冷存储 | SEC-RET-003 |
| 用户注销数据 | 30 天内 | 级联删除 + 匿名化 | SEC-RET-004 |
| 订单数据 | 永久（财务合规） | 归档 | SEC-RET-005 |
| 聊天记录 | 90 天 | 自动删除 | SEC-RET-006 |
| 短信验证码 | 5 分钟（验证码） | 过期自动清理 | SEC-RET-007 |
| 临时文件 | 24 小时 | 定时任务清理 | SEC-RET-008 |
| CI/CD 日志 | 30 天 | 自动清理 | SEC-RET-009 |

### 6.1 用户注销流程

MUST：用户注销后 30 天内完成数据清理。

```text
1. 用户发起注销请求
2. 立即禁用账号（不可登录）
3. 30 天冷静期（可撤销注销）
4. 冷静期结束：
   a. 删除用户表个人信息
   b. 匿名化关联数据（订单保留但移除用户标识）
   c. 删除收货地址、支付信息
   d. 保留匿名统计数据
5. 记录数据清理审计日志
```

## 7. 数据备份策略

> MUST：核心数据必须有定期备份和恢复演练。

| 备份类型 | 频率 | 保留期 | 存储位置 | 加密 |
|----------|------|--------|----------|------|
| 全量备份 | 每日 02:00 | 7 天 | 异地 OSS | AES-256 |
| 增量备份 | 每小时 | 24 小时 | 本地 + 异地 | AES-256 |
| Binlog 备份 | 实时 | 7 天 | 异地 OSS | AES-256 |
| Redis RDB | 每日 03:00 | 3 天 | 本地磁盘 | 无（内网隔离） |
| 配置备份 | 每次变更 | 永久 | Git | N/A |

### 7.1 恢复演练

SHOULD：每季度进行一次备份恢复演练。

```text
演练内容：
1. 从最近备份恢复数据库到测试环境
2. 验证数据完整性和一致性
3. 记录恢复时间（RTO 验证）
4. 演练结果归档
```

## 8. 自动化检查

### 8.1 敏感字段明文检测

CI 中自动检测代码中是否有敏感字段明文输出的风险：

| 检查项 | 规则编号 |
|--------|----------|
| 日志中输出身份证号 | SEC-LOG-001 |
| 日志中输出完整手机号 | SEC-LOG-002 |
| 日志中输出密码/密钥 | SEC-LOG-003 |
| API 响应返回完整银行卡号 | SEC-API-001 |
| 数据库字段未加密存储 L4 数据 | SEC-DB-001 |
