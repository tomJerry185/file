# 中间件规范快速参考卡 | Middleware Quick Reference

> 完整规范见《[中间件规范](中间件规范.md)》，本文档仅列出MUST级别核心规则。

## 通用接入规则

| 规则 | 要求 | 规范章节 |
|------|------|----------|
| 配置管理 | 敏感配置必须来自环境变量，禁止硬编码 | 2.1 |
| 超时配置 | 所有中间件调用必须配超时 | 2.2 |
| 重试策略 | 必须配重试上限+退避策略，仅对幂等操作重试 | 2.2 |
| 连接池 | 必须配上限+空闲回收+健康检查 | 2.2 |
| 环境隔离 | dev/test/staging/prod必须独立实例 | 2.4 |

## Redis（4.5节）

| 规则 | 要求 |
|------|------|
| Key命名 | `{业务}:{模块}:{标识}:{唯一键}` |
| TTL | 业务缓存必须设TTL，雪崩防护+10%随机偏移 |
| 大Key | 禁止>1MB，热点Key需保护 |
| 击穿防护 | 热点Key用互斥锁 |
| 穿透防护 | 不存在数据也缓存NULL + 布隆过滤器 |

## RocketMQ（4.7节）

| 规则 | 要求 |
|------|------|
| 幂等性 | 消费端必须支持重复消息处理，用业务唯一键 |
| 死信队列 | 必须配置死信队列+告警 |
| 发送超时 | 3秒，失败重试2次 |
| 最大重试 | 消费端max-reconsume-times=3 |

## Seata分布式事务（4.8节）

| 规则 | 要求 |
|------|------|
| 事务边界 | 全局事务超时<=60秒，禁止长耗时逻辑 |
| 禁止嵌套 | 分布式事务与MQ不得嵌套 |
| 模式选择 | 简单CRUD用AT模式，复杂业务用TCC模式 |

## Spring Cloud Gateway（4.2节）

| 规则 | 要求 |
|------|------|
| 职责边界 | 禁止业务逻辑/数据库/第三方接口 |
| 统一鉴权 | AuthGlobalFilter + Token校验 |
| TraceId | TraceIdGlobalFilter最先执行 |
| 响应式编程 | 禁止subscribe()后立即返回成功 |

## MySQL（4.9节）

| 规则 | 要求 |
|------|------|
| 慢查询阈值 | > 1秒，必须告警 |
| 分页限制 | 最大<=1000，深分页用游标 |
| 索引 | WHERE/ORDER BY/GROUP BY列必须有索引 |

## 关键配置速查

### 超时配置

| 中间件 | 连接超时 | 读取超时 |
|--------|----------|----------|
| HTTP/Feign | 3s | 30s |
| Redis | 2s | 5s |
| MySQL | 5s | 30s |
| RocketMQ | 3s | - |
| ES | 5s | 60s |

### 连接池配置

| 中间件 | 最小连接 | 最大连接 | HikariCP说明 |
|--------|----------|----------|-------------|
| MySQL | 5 | 20（高并发50） | idle 10min |
| Redis | 5 | 50 | idle 5min |
| HTTP | 5 | 200 | idle 60s |

### HikariCP（通用服务）

```yaml
spring.datasource.hikari:
  minimum-idle: 5
  maximum-pool-size: 20  # 高并发服务: 50
  connection-timeout: 30000
  idle-timeout: 600000
  max-lifetime: 1800000
```
