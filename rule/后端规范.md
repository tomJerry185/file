0. 基本原则与术语表 | General Principles & Glossary
================================================================================
0.1 规范分级说明 | Specification Levels
- MUST（必须）：强制要求，违反将导致质量/安全/稳定性风险
- SHOULD（应该）：强烈推荐，例外需审批和文档记录
- MAY（可以）：可选建议，根据场景自行决定
- MUST NOT（禁止）：严格禁止，违反将触发违规处理

0.2 适用范围 | Scope
- 适用于所有后端项目：Spring Boot / Spring Cloud / 微服务
- 适用于所有环境：开发/测试/预发/生产
- 适用于所有语言：Java / Kotlin / Groovy（以Java为主）

0.3 例外与豁免机制 | Exceptions & Waivers
- 例外必须经后端架构组审批
- 例外必须包含：原因/范围/风险/有效期/回滚方案/追踪编号
- 例外到期必须复审或清理
- 验证方式：例外审批记录/RFC

0.4 违规处理 | Violation Handling
- MUST级别违规：立即修复，并提交RCA
- SHOULD级别违规：5个工作日内修复或申请例外
- MAY级别违规：建议采纳，无强制要求
- 验证方式：违规跟踪系统/代码审查

0.5 术语表 | Glossary

> **通用术语**（代码审查、回滚、SLA/SLO、链路追踪、服务网格等）统一定义在《工程治理规范.md》第19.7节"术语表（完整版）"，此处仅列出后端领域专有术语。

1. 架构设计规范（MUST）| Architecture Design Standards
================================================================================
1.1 架构原则 | Architecture Principles
MUST 1.1.01 单一职责原则（SRP）
- 一个类/模块/服务只负责一个职责
- 验证方式：代码审查

MUST 1.1.02 开放封闭原则（OCP）
- 对扩展开放，对修改封闭
- 通过抽象和多态实现
- 验证方式：代码审查

MUST 1.1.03 依赖倒置原则（DIP）
- 高层模块不依赖低层模块，两者都依赖抽象
- 抽象不依赖具体，具体依赖抽象
- 验证方式：架构审查

MUST 1.1.04 接口隔离原则（ISP）
- 客户端不应依赖它不需要的接口
- 接口应细粒度，职责单一
- 验证方式：代码审查

MUST 1.1.05 最少知识原则（LoD）/迪米特法则
- 对象之间应最少交互
- 只与直接朋友通信
- 验证方式：代码审查

1.2 分层架构模式 | Layered Architecture Pattern
MUST 1.2.01 推荐分层结构
```
┌─────────────────────────────────────────────────────┐
│                   API Gateway Layer                 │
│         （路由、认证、限流、熔断、监控）              │
└─────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────┐
│              Presentation Layer (Controller)          │
│              （参数校验、权限校验、请求编排）         │
└─────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────┐
│                 Application Layer (Service)           │
│              （业务逻辑、事务编排、领域服务）         │
└─────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────┐
│                  Domain Layer (Domain)                │
│         （领域模型、领域服务、领域事件、仓储接口）     │
└─────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────┐
│               Infrastructure Layer (Impl)              │
│              （数据访问、外部服务、技术实现）         │
└─────────────────────────────────────────────────────┘
```

MUST 1.2.02 层级依赖规则
- 上层可以依赖下层，下层不能依赖上层
- 同层之间通过接口交互
- 跨层调用禁止（Controller不能直接调用Mapper）
- 验证方式：架构扫描工具/代码审查

1.3 DDD架构模式 | Domain-Driven Design Pattern
MUST 1.3.01 战略设计层
- 定义限界上下文（Bounded Context）
- 定义上下文映射（Context Mapping）
- 验证方式：架构文档/架构审查

MUST 1.3.02 战术设计层
- Entity：实体（有唯一标识）
- Value Object：值对象（不可变）
- Aggregate：聚合根（一致性边界）
- Repository：仓储接口
- Domain Service：领域服务
- Domain Event：领域事件
- 验证方式：代码审查

1.4 设计模式使用规范 | Design Patterns Usage
MUST 1.4.01 推荐使用的设计模式
- **工厂模式（Factory）**：对象创建复杂
- **策略模式（Strategy）**：算法族
- **模板方法模式（Template Method）**：流程固定，步骤可变
- **观察者模式（Observer）**：事件驱动
- **责任链模式（Chain of Responsibility）**：审批流程
- **装饰器模式（Decorator）**：动态增强功能
- 验证方式：代码审查

MUST 1.4.02 Spring框架内置模式优先
- 使用Spring的依赖注入替代手动实现
- 使用Spring事件机制（ApplicationEvent）替代Observer
- 使用Spring模板（JdbcTemplate/RestTemplate）
- 验证方式：代码审查

2. 分层架构规范（MUST）| Layered Architecture Standards
================================================================================
2.1 Controller层 | Controller Layer
MUST 2.1.01 Controller职责
- 参数校验（Bean Validation）
- 权限校验（调用授权服务）
- 调用Service编排业务逻辑
- 返回统一响应结构
- 禁止包含业务逻辑
- 禁止访问数据库
- 验证方式：代码审查

MUST 2.1.02 Controller命名
- 以`*Controller`结尾
- 以功能模块命名：`UserController`/`OrderController`
- 验证方式：命名检查

```java
// 正确示例 ✓
@RestController
@RequestMapping("/api/users")
public class UserController {

    @GetMapping("/{id}")
    public Result<UserVO> getUser(@PathVariable Long id) {
        return Result.success(userService.getUserById(id));
    }
}

// 错误示例 ✗
@Controller
public class UserMgr {  // 命名不规范
    @GetMapping("/user/{id}")
    public User getUser(@PathVariable Long id) {  // 返回Entity，未包装
        return userRepository.findById(id);  // 直接访问Repository
    }
}
```

MUST 2.1.03 统一响应结构
```java
/**
 * 统一响应结构
 * @param <T> 数据类型
 */
public class Result<T> {
    private Integer code;
    private String message;
    private T data;
    private String traceId;
    private Long timestamp;

    // 成功响应
    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setMessage("success");
        result.setData(data);
        result.setTraceId(MDC.get("traceId"));
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }

    // 失败响应
    public static <T> Result<T> error(Integer code, String message) {
        Result<T> result = new Result<>();
        result.setCode(code);
        result.setMessage(message);
        result.setTraceId(MDC.get("traceId"));
        result.setTimestamp(System.currentTimeMillis());
        return result;
    }

    // 分页响应
    public static <T> Result<PageInfo<T>> page(Page<T> page) {
        return success(PageInfo.of(page));
    }
}
```

2.2 Service层 | Service Layer
MUST 2.2.01 Service接口规范
- 所有Service必须定义接口
- 接口命名：`*Service`
- 实现类命名：`*ServiceImpl`
- 验证方式：命名检查

MUST 2.2.02 Service职责
- 业务逻辑处理
- 事务管理
- 领域模型转换
- 外部服务调用
- 验证方式：代码审查

MUST 2.2.03 事务边界
```java
@Service
@Transactional(readOnly = true)
public class OrderServiceImpl implements OrderService {

    @Transactional(rollbackFor = Exception.class, timeout = 30)
    public Order createOrder(CreateOrderCmd cmd) {
        // 1. 参数校验
        validateCreateOrder(cmd);

        // 2. 业务逻辑
        Order order = buildOrder(cmd);

        // 3. 持久化
        orderMapper.insert(order);

        // 4. 发布领域事件
        eventPublisher.publish(new OrderCreatedEvent(order.getId()));

        return order;
    }
}
```

MUST 2.2.04 只读事务优化
- 查询方法使用`@Transactional(readOnly = true)`
- 避免不必要的写锁
- 验证方式：代码审查

2.3 Mapper/DAO层 | Mapper/DAO Layer
MUST 2.3.01 Mapper接口规范
- 继承`BaseMapper<T>`（MyBatis-Plus）
- 命名：`*Mapper`
- 验证方式：命名检查

```java
// 正确示例 ✓
@Mapper
public interface UserMapper extends BaseMapper<UserDO> {

    /**
     * 根据用户名查询用户
     * @param username 用户名
     * @return 用户DO，不存在返回null
     */
    UserDO selectByUsername(@Param("username") String username);
}

// 错误示例 ✗
@Mapper
public interface UserDAO {  // 命名不规范
    User findUser(String name);  // 方法命名不规范
}
```

MUST 2.3.02 禁止在Mapper中拼接SQL
- 必须使用注解或XML定义SQL
- 必须使用`#{}`参数化
- 验证方式：SQL扫描

3. RESTful API设计规范（MUST）| RESTful API Standards
================================================================================
3.1 URL设计 | URL Design
MUST 3.1.01 URL命名规范
- 使用小写字母
- 使用连字符`-`分隔单词
- 使用复数名词表示资源
- 避免使用动词，使用HTTP方法表示操作

```
正确示例 ✓:
GET    /api/users           # 获取用户列表
GET    /api/users/{id}      # 获取单个用户
POST   /api/users           # 创建用户
PUT    /api/users/{id}      # 更新用户
PATCH  /api/users/{id}      # 部分更新
DELETE /api/users/{id}      # 删除用户
GET    /api/users/{id}/orders # 获取用户的订单

错误示例 ✗:
GET    /api/getUsers         # 使用动词
GET    /api/user/{id}       # 使用单数
POST   /api/createUser      # 使用动词
GET    /api/listUsers       # 使用动词
```

MUST 3.1.02 查询参数规范
```
分页: ?page=1&size=20&sort=createdAt:desc
过滤: ?status=ACTIVE&type=VIP
搜索: ?keyword=iphone&category=phone
范围: ?priceMin=1000&priceMax=5000&dateFrom=2024-01-01
```

3.2 HTTP方法使用 | HTTP Methods
MUST 3.2.01 方法语义
```
GET    - 查询资源，幂等、安全
POST   - 创建资源，非幂等
PUT    - 全量更新资源，幂等
PATCH  - 部分更新资源，幂等
DELETE - 删除资源，幂等
```

MUST 3.2.02 状态码规范
```
200 OK              - 成功
201 Created         - 创建成功
204 No Content      - 删除成功/更新无返回
400 Bad Request     - 请求参数错误
401 Unauthorized    - 未认证
403 Forbidden       - 无权限
404 Not Found       - 资源不存在
409 Conflict        - 资源冲突
422 Unprocessable  - 业务逻辑错误
429 Too Many Requests - 限流
500 Internal Error  - 服务器错误
503 Service Unavailable - 服务不可用
```

3.3 API版本控制 | API Versioning
MUST 3.3.01 版本控制策略
- 使用URL路径版本：`/api/v1/users`
- 或使用请求头版本：`Accept: application/vnd.api.v1+json`
- 验证方式：API审查

4. Controller层规范（MUST）| Controller Layer Standards
================================================================================
4.1 参数校验 | Parameter Validation
MUST 4.1.01 Bean Validation
```java
// DTO定义
public class CreateUserDTO {

    @NotBlank(message = "用户名不能为空")
    @Length(min = 2, max = 20, message = "用户名长度2-20字符")
    private String username;

    @NotBlank(message = "邮箱不能为空")
    @Email(message = "邮箱格式不正确")
    private String email;

    @NotBlank(message = "手机号不能为空")
    @Pattern(regexp = "^1[3-9]\\d{9}$", message = "手机号格式不正确")
    private String phone;

    @NotNull(message = "类型不能为空")
    private UserType userType;
}

// Controller使用
@PostMapping
public Result<UserVO> create(@Valid @RequestBody CreateUserDTO dto) {
    // ...
}
```

MUST 4.1.02 自定义校验器
```java
@Target({ElementType.FIELD})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneValidator.class)
public @interface Phone {
    String message() default "手机号格式不正确";
    Class<?>[] groups() default {};
}

public class PhoneValidator implements ConstraintValidator<Phone, String> {
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        return value != null && value.matches("^1[3-9]\\d{9}$");
    }
}
```

4.2 禁止占位实现 | No Placeholder Implementation
MUST 4.2.01 禁止返回假数据
```java
// 禁止模式 ✗
@GetMapping("/users/{id}")
public Result<UserVO> getUser(@PathVariable Long id) {
    return Result.success(new UserVO());  // 空对象，伪数据
}

// 禁止模式 ✗
@GetMapping("/list")
public Result<List<UserVO>> list() {
    return Result.success(Collections.emptyList());  // 空列表，伪装成功
}

// 正确模式 ✓
@GetMapping("/users/{id}")
public Result<UserVO> getUser(@PathVariable Long id) {
    UserVO user = userService.getUserById(id);
    if (user == null) {
        return Result.error(404, "用户不存在");
    }
    return Result.success(user);
}
```

5. Service层规范（MUST）| Service Layer Standards
================================================================================
5.1 依赖注入 | Dependency Injection
MUST 5.1.01 构造器注入优先
```java
// 正确示例 ✓ - 构造器注入
@Service
public class OrderServiceImpl implements OrderService {

    private final OrderMapper orderMapper;
    private final ProductService productService;
    private final UserService userService;

    public OrderServiceImpl(OrderMapper orderMapper,
                            ProductService productService,
                            UserService userService) {
        this.orderMapper = orderMapper;
        this.productService = productService;
        this.userService = userService;
    }
}

// 错误示例 ✗ - 字段注入
@Service
public class OrderServiceImpl implements OrderService {

    @Autowired
    private OrderMapper orderMapper;  // 字段注入，不利于测试
}

// 错误示例 ✗ - Setter注入
@Service
public class OrderServiceImpl implements OrderService {

    private OrderMapper orderMapper;

    @Autowired
    public void setOrderMapper(OrderMapper orderMapper) {
        this.orderMapper = orderMapper;  // Setter注入，依赖可能不完整
    }
}
```

MUST 5.1.02 Lombok推荐使用
```java
// 推荐 ✓
@Service
@RequiredArgsConstructor // Lombok生成构造器
public class OrderServiceImpl implements OrderService {
    private final OrderMapper orderMapper;
    private final ProductService productService;
}
```

5.2 事务管理 | Transaction Management
MUST 5.2.01 事务边界
```java
@Service
public class OrderServiceImpl implements OrderService {

    // 正确：新事务
    @Transactional(propagation = Propagation.REQUIRES_NEW, timeout = 30)
    public void createOrder(Order order) {
        // ...
    }

    // 正确：只读事务
    @Transactional(readOnly = true)
    public Order getOrder(Long id) {
        // ...
    }

    // 错误：过大的事务 ✗
    @Transactional
    public void processBigData() {
        // 处理大量数据，可能导致锁超时
    }
}
```

MUST 5.2.02 事务传播行为
```
REQUIRED（默认）    - 加入当前事务，无事务则创建
REQUIRES_NEW       - 创建新事务，挂起当前事务
NESTED             - 嵌入当前事务
SUPPORTS           - 支持当前事务，无事务则非事务执行
NOT_SUPPORTED      - 非事务执行
MANDATORY          - 必须在事务中执行
NEVER              - 非事务执行，抛异常
```

6. Mapper/DAO层规范（MUST）| Mapper/DAO Layer Standards
================================================================================
6.1 SQL规范 | SQL Standards
MUST 6.1.01 禁止SELECT *
```java
// 正确示例 ✓
@Select("SELECT id, username, email, status FROM t_user WHERE id = #{id}")
UserDO selectById(@Param("id") Long id);

// 错误示例 ✗
@Select("SELECT * FROM t_user WHERE id = #{id}")
UserDO selectById(@Param("id") Long id);
```

MUST 6.1.02 分页查询规范
```java
// 正确示例 ✓ - 使用PageHelper或IPage
Page<UserDO> selectPage(Page<UserDO> page, @Param("keyword") String keyword);

// XML
<select id="selectPage" resultType="UserDO">
    SELECT id, username, email, status
    FROM t_user
    <where>
        <if test="keyword != null and keyword != ''">
            AND username LIKE CONCAT('%', #{keyword}, '%')
        </if>
    </where>
    ORDER BY id DESC
</select>
```

MUST 14.1.01 枚举类强制使用原则
```
【强制要求】所有状态码和错误码必须使用枚举(Enum)类实现

禁止使用：
1. 魔法数字（如 return 1; return -1;）
2. 字符串常量（如 public static final String SUCCESS = "200";）
3. 整数常量接口（如 interface Code { int SUCCESS = 200; }）

原因：
- 枚举提供编译时类型安全
- 枚举支持反向查询（code -> enum）
- 枚举可携带额外信息（详细原因、解决建议）
- IDE自动补全，减少拼写错误
- 便于统一管理和重构
```

```java
/**
 * 业务异常基类
 * 所有业务异常必须继承此类
 */
public class BusinessException extends RuntimeException {

    private static final long serialVersionUID = 1L;

    /** 错误码（必须） */
    private final int code;

    /** 错误消息（必须） */
    private final String message;

    /** 详细错误原因（可选） */
    private final String detailedReason;

    /** 解决建议（可选） */
    private final String suggestion;

    /** 动态详情（可选，用于传递额外上下文） */
    private final Map<String, Object> details;

    // ========== 推荐构造函数：使用ErrorCodeEnum ==========
    /**
     * 使用错误码枚举构造（推荐）
     */
    public BusinessException(ErrorCodeEnum errorCode) {
        super(errorCode.getMessage());
        this.code = errorCode.getCode();
        this.message = errorCode.getMessage();
        this.detailedReason = errorCode.getDetailedReason();
        this.suggestion = errorCode.getSuggestion();
        this.details = null;
    }

    /**
     * 使用错误码枚举和自定义消息
     */
    public BusinessException(ErrorCodeEnum errorCode, String customMessage) {
        super(customMessage);
        this.code = errorCode.getCode();
        this.message = customMessage;
        this.detailedReason = errorCode.getDetailedReason();
        this.suggestion = errorCode.getSuggestion();
        this.details = null;
    }

    /**
     * 使用错误码枚举和原始异常（保留异常链）
     */
    public BusinessException(ErrorCodeEnum errorCode, Throwable cause) {
        super(errorCode.getMessage(), cause);
        this.code = errorCode.getCode();
        this.message = errorCode.getMessage();
        this.detailedReason = errorCode.getDetailedReason();
        this.suggestion = errorCode.getSuggestion();
        this.details = null;
    }

    /**
     * 完整构造：错误码枚举 + 自定义消息 + 原始异常 + 动态详情
     */
    public BusinessException(ErrorCodeEnum errorCode, String customMessage,
                            Throwable cause, Map<String, Object> details) {
        super(customMessage, cause);
        this.code = errorCode.getCode();
        this.message = customMessage;
        this.detailedReason = errorCode.getDetailedReason();
        this.suggestion = errorCode.getSuggestion();
        this.details = details != null ? new HashMap<>(details) : null;
    }

    // ========== 兼容构造函数：直接使用code和message ==========
    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
        this.message = message;
        this.detailedReason = null;
        this.suggestion = null;
        this.details = null;
    }

    public BusinessException(int code, String message, Throwable cause) {
        super(message, cause);
        this.code = code;
        this.message = message;
        this.detailedReason = null;
        this.suggestion = null;
        this.details = null;
    }

    // ========== Getter方法 ==========
    public int getCode() { return code; }
    @Override public String getMessage() { return message; }
    public String getDetailedReason() { return detailedReason; }
    public String getSuggestion() { return suggestion; }
    public Map<String, Object> getDetails() {
        return details != null ? new HashMap<>(details) : null;
    }

    public String getFullMessage() {
        return "[" + code + "] " + message;
    }
}
```

MUST 14.2.02 异常使用规范
```java
@Service
@RequiredArgsConstructor
public class OrderServiceImpl implements OrderService {

    private final OrderMapper orderMapper;

    // ========== 正确示例1：直接抛出枚举异常 ==========
    public Order getOrderById(Long orderId) {
        return orderMapper.selectById(orderId)
            .orElseThrow(() -> new BusinessException(ErrorCodeEnum.ORDER_NOT_FOUND));
    }

    // ========== 正确示例2：带自定义消息 ==========
    public void cancelOrder(Long orderId, Long userId) {
        Order order = orderMapper.selectById(orderId)
            .orElseThrow(() -> new BusinessException(ErrorCodeEnum.ORDER_NOT_FOUND,
                "订单 " + orderId + " 不存在"));

        if (!order.getUserId().equals(userId)) {
            throw new BusinessException(ErrorCodeEnum.PERMISSION_DENIED,
                "您无权取消此订单");
        }
    }

    // ========== 正确示例3：保留原始异常链 ==========
    public void updateOrderStatus(Long orderId, OrderStatus status) {
        try {
            orderMapper.updateStatus(orderId, status);
        } catch (DataAccessException e) {
            // 使用异常链，保留原始异常用于问题排查
            throw new BusinessException(
                ErrorCodeEnum.DATABASE_ERROR,
                "更新订单状态失败",
```

```java
/**
 * 异常处理工具类
 * 提供统一的异常转换和处理方法
 */
public final class ExceptionUtils {

    private ExceptionUtils() {}

    /**
     * 将底层异常转换为业务异常（保留异常链）
     */
    public static BusinessException wrapBusinessException(
            ErrorCodeEnum errorCode, Throwable cause) {
        return new BusinessException(errorCode, cause.getMessage(), cause);
    }

    /**
     * 将SQL异常转换为业务异常
     * 根据SQLState判断具体错误类型
     */
    public static BusinessException fromSqlException(SQLException e) {
        String state = e.getSQLState();

        // 主键冲突
        if ("23000".equals(state) || e.getErrorCode() == 1062) {
            return new BusinessException(
                ErrorCodeEnum.DATABASE_DUPLICATE_KEY,
                "数据已存在，请勿重复添加",
                e
            );
        }

        // 连接失败
        if ("08001".equals(state)) {
            return new BusinessException(
                ErrorCodeEnum.DATABASE_ERROR,
                "数据库连接失败",
                e
            );
        }

        // 默认数据库错误
        return new BusinessException(
            ErrorCodeEnum.DATABASE_ERROR,
            "数据库操作失败",
            e
        );
    }

    /**
     * 获取异常的根本原因
     * 遍历异常链找到最底层的cause
     */
    public static Throwable getRootCause(Throwable throwable) {
        Throwable cause = throwable;
        while (cause.getCause() != null) {
            cause = cause.getCause();
        }
        return cause;
    }

    /**
     * 判断异常是否由指定类型导致
     */
    public static boolean isCausedBy(Throwable throwable, Class<?> causeType) {
        Throwable current = throwable;
        while (current != null) {
            if (causeType.isInstance(current)) {
                return true;
            }
            current = current.getCause();
        }
        return false;
    }
}
```

```java
/**
 * 全局异常处理器
 * 统一处理所有异常并返回标准格式响应
 */
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    /**
     * 处理业务异常
     */
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<Result<Void>> handleBusinessException(
            BusinessException ex, WebRequest request) {

        log.warn("Business exception: [{}] {}",
            ex.getCode(), ex.getMessage());

        Result<Void> result = Result.error(ex.getCode(), ex.getMessage());
        result.setDetailedReason(ex.getDetailedReason());
        result.setSuggestion(ex.getSuggestion());
        result.setTraceId(MDC.get("traceId"));

        return ResponseEntity
            .status(getHttpStatusFromErrorCode(ex.getCode()))
            .body(result);
    }

    /**
     * 处理参数校验异常
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Result<Void>> handleValidationException(
            MethodArgumentNotValidException ex) {

        String message = ex.getBindingResult().getFieldErrors().stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.joining("; "));

        log.warn("Validation failed: {}", message);

        return ResponseEntity
            .status(HttpStatus.BAD_REQUEST)
            .body(Result.error(ErrorCodeEnum.BAD_REQUEST.getCode(), message));
    }

    /**
     * 处理未知异常
     */
    @ExceptionHandler(Exception.class)
    public ResponseEntity<Result<Void>> handleUnhandledException(
            Exception ex, WebRequest request) {

        log.error("Unhandled exception", ex);

        String errorId = UUID.randomUUID().toString();
        MDC.put("errorId", errorId);

        Result<Void> result = Result.error(ErrorCodeEnum.INTERNAL_ERROR);
        result.setErrorId(errorId);
        result.setTraceId(MDC.get("traceId"));

        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(result);
    }

    /**
     * 根据错误码获取HTTP状态码
     */
    private int getHttpStatusFromErrorCode(int errorCode) {
        if (errorCode >= 200 && errorCode < 300) return 200;
        if (errorCode >= 400 && errorCode < 500) return errorCode;
        return 500;
    }
}
```

20.1.2 logback-spring.xml
```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <springProperty scope="context" name="activeProfile" source="spring.profiles.active" defaultValue="dev"/>

    <property name="LOG_PATH" value="logs"/>
    <property name="APP_NAME" value="user-service"/>

    <!-- 控制台输出 -->
    <appender name="CONSOLE" class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level [%X{traceId:-}] %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 文件输出 -->
    <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
        <file>${LOG_PATH}/${APP_NAME}.log</file>
        <rollingPolicy class="ch.qos.logback.core.rolling.SizeAndTimeBasedRollingPolicy">
            <fileNamePattern>${LOG_PATH}/${APP_NAME}-%d{yyyy-MM-dd}.%i.log</fileNamePattern>
            <maxFileSize>100MB</maxFileSize>
            <maxHistory>30</maxHistory>
            <totalSizeCap>10GB</totalSizeCap>
        </rollingPolicy>
        <encoder>
            <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level [%X{traceId:-}] %logger{36} - %msg%n</pattern>
            <charset>UTF-8</charset>
        </encoder>
    </appender>

    <!-- 异步日志 -->
    <appender name="ASYNC_FILE" class="ch.qos.logback.classic.AsyncAppender">
        <appender-ref ref="FILE"/>
        <queueSize>512</queueSize>
        <discardingThreshold>0</discardingThreshold>
    </appender>

    <!-- 日志级别 -->
    <root level="INFO">
        <appender-ref ref="CONSOLE"/>
        <appender-ref ref="ASYNC_FILE"/>
    </root>

    <!-- 特定日志级别 -->
    <logger name="com.example.users" level="DEBUG"/>
    <logger name="org.springframework.web" level="INFO"/>
    <logger name="org.hibernate.SQL" level="WARN"/>

    <!-- 开发环境 -->
    <springProfile name="dev">
        <root level="DEBUG">
            <appender-ref ref="CONSOLE"/>
            <appender-ref ref="ASYNC_FILE"/>
        </root>
    </springProfile>

    <!-- 生产环境 -->
    <springProfile name="prod">
        <root level="INFO">
            <appender-ref ref="ASYNC_FILE"/>
        </root>
        <logger name="com.example.users" level="INFO"/>
    </springProfile>
</configuration>
```

20.1.3 Dockerfile
```dockerfile
FROM openjdk:17-jdk-slim

LABEL maintainer="backend-team@example.com"

WORKDIR /app

# 复制Maven配置文件
COPY pom.xml .
COPY mvnw .

# 复制源码
COPY src ./src

# 构建应用
RUN mvn clean package -DskipTests -Dmaven.test.skip=true

# 运行时参数
ENV JAVA_OPTS=""
ENV SERVER_PORT=8080

# 暴露端口
EXPOSE 8080

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s \
    CMD curl -f http://localhost:8080/actuator/health || exit 1

# 启动应用
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -jar /app/app.jar"]
```

20.1.4 docker-compose.yml
```yaml
services:
  user-service:
    build: .
    container_name: user-service
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_URL=jdbc:mysql://mysql:3306/user_service?useSSL=false&serverTimezone=Asia/Shanghai
      - REDIS_HOST=redis
      - REDIS_PORT=6379
      - NACOS_ADDR=nacos:8848
      - SENTINEL_ADDR=sentinel:8080
      - ROCKETMQ_ADDR=rocketmq-nameserver:9876
    depends_on:
      - mysql
      - redis
      - nacos
      - rocketmq-nameserver
    networks:
      - microservice
    restart: unless-stopped

  mysql:
    image: mysql:8.0
    container_name: mysql
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}  # ⚠️ 生产环境禁止硬编码密码，必须使用环境变量或密钥管理工具
      - MYSQL_DATABASE=user_service
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - microservice
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    container_name: redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - microservice
    restart: unless-stopped

  nacos:
    image: nacos/nacos-server:v2.2.3
    container_name: nacos
    ports:
      - "8848:8848"
      - "9848:9848"
    environment:
      - MODE=standalone
    volumes:
      - nacos-data:/home/nacos/data
    networks:
      - microservice
    restart: unless-stopped

  sentinel:
    image: bladex/sentinel-dashboard:1.8.6
    container_name: sentinel
    ports:
      - "8080:8080"
    networks:
      - microservice
    restart: unless-stopped

  rocketmq-nameserver:
    image: apache/rocketmq:5.1.4
    container_name: rocketmq-nameserver
    ports:
      - "9876:9876"
    command: sh mqnamesrv
    networks:
      - microservice
    restart: unless-stopped

  rocketmq-broker:
    image: apache/rocketmq:5.1.4
    container_name: rocketmq-broker
    ports:
      - "10911:10911"
    environment:
      - NAMESRV_ADDR=rocketmq-nameserver:9876
    command: sh mqbroker -n rocketmq-nameserver:9876
    depends_on:
      - rocketmq-nameserver
    networks:
      - microservice
    restart: unless-stopped

volumes:
  mysql-data:
  redis-data:
  nacos-data:

networks:
  microservice:
    driver: bridge
```
