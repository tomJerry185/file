## 1. API 限流防护

> MUST：所有面向外部用户的 API 必须配置限流，防止资源耗尽攻击。

### 1.1 限流阈值表

| 接口类别 | 限流维度 | 阈值 | 超限响应 | 规则编号 |
|----------|----------|------|----------|----------|
| 登录/注册 | IP | 5次/分钟 | HTTP 429 + 提示"操作过于频繁，请X秒后重试" | SEC-RATE-001 |
| 登录/注册 | 用户+IP | 10次/分钟 | HTTP 429 | SEC-RATE-002 |
| 下单/支付 | 用户 | 10次/分钟 | HTTP 429 + 降级提示 | SEC-RATE-003 |
| 搜索/查询 | 用户 | 100次/分钟 | HTTP 429 | SEC-RATE-004 |
| 搜索/查询 | IP | 200次/分钟 | HTTP 429 | SEC-RATE-005 |
| 短信验证码 | 手机号 | 1次/60秒 | HTTP 429 | SEC-RATE-006 |
| 短信验证码 | IP | 5次/小时 | HTTP 429 | SEC-RATE-007 |
| 文件上传 | 用户 | 10次/分钟 | HTTP 429 | SEC-RATE-008 |
| AI 对话 | 用户 | 20次/分钟 | HTTP 429 | SEC-RATE-009 |
| 通用 API | IP | 1000次/分钟 | HTTP 429 | SEC-RATE-010 |

### 1.2 限流实现规范

MUST：使用 Sentinel 或 Redis + Lua 实现限流，禁止在应用层使用单机内存限流。

MUST：限流响应必须包含以下信息：

```json
{
  "code": 429,
  "message": "操作过于频繁，请30秒后重试",
  "retryAfter": 30
}
```

MUST：关键业务接口（支付、下单）限流失败后必须记录日志：
- 日志级别：WARN
- 包含：用户ID、IP、接口路径、当前QPS

### 1.3 限流配置位置

| 配置项 | 位置 | 说明 |
|--------|------|------|
| Sentinel 流控规则 | Nacos 配置中心 | `rule/automation/prometheus-alerts.yml` 同步 |
| API Gateway 全局限流 | `api-gateway` 模块 | Redis Token Bucket |
| 业务级限流 | 各微服务 | Sentinel `@SentinelResource` |

## 2. 暴力破解防护

> MUST：所有认证接口必须实现防暴力破解机制。

### 2.1 防护策略

| 维度 | 规则 | 锁定时间 | 规则编号 |
|------|------|----------|----------|
| IP 维度 | 同一 IP 连续失败 N 次 | 锁定 M 分钟 | SEC-BF-001 |
| 用户维度 | 同一账号连续失败 N 次 | 锁定 M 分钟 + 通知用户 | SEC-BF-002 |
| 图形验证码 | 失败 3 次后启用 | 直至成功或超时 | SEC-BF-003 |
| 短信验证码 | 频率限制 + 验证失败 5 次 | 手机号锁定 30 分钟 | SEC-BF-004 |

### 2.2 实现要求

MUST：失败计数使用 Redis，设置合理的 TTL（与锁定时间一致）。

MUST：账号锁定时发送安全通知（短信/邮件），告知用户非本人操作可联系客服。

MUST：管理员可通过后台手动解除锁定，操作需记录审计日志。

### 2.3 默认参数

```text
IP 维度：连续失败 10 次 → 锁定 30 分钟
用户维度：连续失败 5 次 → 锁定 15 分钟 + 短信通知
图形验证码：失败 3 次后启用
验证码有效期：5 分钟
验证码尝试次数：最多 3 次
```

## 3. CSRF 防护

> MUST：所有状态变更接口（POST/PUT/DELETE）必须实施 CSRF 防护。

### 3.1 需要防护的接口

| 接口类型 | CSRF 防护 | 说明 |
|----------|-----------|------|
| POST/PUT/DELETE | MUST | 状态变更操作 |
| GET | 不需要 | 幂等查询操作 |
| 文件上传 | MUST | 使用 multipart + token |
| API（Bearer Token 认证） | 自动 | 同源策略 + CORS 够用 |
| 表单提交（Cookie 认证） | MUST | 需要 CSRF Token |

### 3.2 CSRF Token 机制

后端（Spring Security 配置）：

```java
// MUST: 启用 CSRF（Cookie 认证场景）
http.csrf(csrf -> csrf
    .csrfTokenRepository(CookieCsrfTokenRepository.withHttpOnlyFalse())
    .csrfTokenRequestHandler(new CsrfTokenRequestAttributeHandler())
);

// MUST: API 场景（Bearer Token）可禁用 CSRF，但必须启用 CORS 白名单
```

前端传递方式：
- 方式一：Header `X-CSRF-TOKEN`（推荐）
- 方式二：Cookie `XSRF-TOKEN`（自动）
- 方式三：表单隐藏字段 `<input type="hidden" name="_csrf">`

### 3.3 CORS 配置

MUST：CORS 只允许白名单域名，禁止 `*`。

```text
生产环境白名单：
- https://www.example.com
- https://admin.example.com
- https://m.example.com

禁止配置：
- Access-Control-Allow-Origin: *
```

## 4. 文件上传安全

> MUST：所有文件上传功能必须实施安全检查。

### 4.1 白名单扩展名

| 类别 | 允许扩展名 | 说明 |
|------|-----------|------|
| 图片 | jpg, jpeg, png, gif, webp, svg | svg 需额外净化 |
| 文档 | pdf, doc, docx, xls, xlsx | 禁止宏文件 |
| 压缩包 | zip, rar, 7z | 需解压扫描 |
| 视频 | mp4, avi, mov | 限制大小 |

MUST 禁止上传的文件类型：
- 可执行文件：exe, bat, sh, cmd, ps1, jar, class
- 脚本文件：php, jsp, asp, aspx, cgi
- 配置文件：htaccess, web.config, .env
- 其他：任何包含双扩展名的文件（如 file.php.jpg）

### 4.2 文件大小限制

| 类型 | 最大大小 | 说明 |
|------|----------|------|
| 头像 | 2 MB | 正方形，自动裁剪 |
| 商品图 | 5 MB | 自动压缩 |
| 文档附件 | 20 MB | 生产环境限制 |
| 视频文件 | 100 MB | 分片上传 |
| 任何文件 | 50 MB | Spring Boot 全局限制 |

### 4.3 安全检查流程

```text
1. 扩展名白名单校验（MUST）
2. 文件大小校验（MUST）
3. Magic Number 校验 — 检查文件头与扩展名是否匹配（MUST）
4. 文件名净化 — 移除特殊字符，使用 UUID 重命名（MUST）
5. 存储路径隔离 — 按日期/类型分目录存储（MUST）
6. 病毒扫描（生产环境 SHOULD）
7. 图片 EXIF 信息剥离（SHOULD）
```

### 4.4 存储安全

MUST：上传文件不存放在 Web 可直接访问的目录下（或配置 Nginx 禁止执行）。

MUST：文件 URL 使用签名 URL，设置过期时间。

SHOULD：图片使用 CDN + 签名访问。

## 5. SQL 注入防护

> MUST：所有数据库查询必须使用参数化查询，禁止字符串拼接 SQL。

### 5.1 基本规则

| 规则 | 级别 | 规则编号 |
|------|------|----------|
| 所有 SQL 必须使用参数化查询（`#{}` / PreparedStatement） | MUST | SEC-SQL-001 |
| 禁止使用 `` `${}` `` 拼接用户输入 | MUST | SEC-SQL-002 |
| 禁止 MyBatis 中使用 `` `${}` `` 传递用户输入 | MUST | SEC-SQL-003 |
| ORDER BY 必须使用白名单校验 | MUST | SEC-SQL-004 |
| LIKE 查询必须使用 `` CONCAT('%', #{keyword}, '%') `` | MUST | SEC-SQL-005 |
| IN 查询必须使用 `<foreach>` 标签 | MUST | SEC-SQL-006 |
| 禁止直接拼接表名/列名 | MUST | SEC-SQL-007 |

### 5.2 特殊场景处理

**ORDER BY 动态排序**：

```java
// MUST: 白名单校验
private static final Set<String> ALLOWED_ORDER_COLUMNS = Set.of(
    "create_time", "update_time", "price", "name", "id"
);

public void validateOrderBy(String column) {
    if (!ALLOWED_ORDER_COLUMNS.contains(column.toLowerCase())) {
        throw new IllegalArgumentException("非法排序字段: " + column);
    }
}
```

**LIKE 模糊查询**：

```xml
<!-- MUST: 使用 CONCAT 防注入 -->
<select id="searchByName" resultType="Product">
    SELECT id, name, price FROM product
    WHERE name LIKE CONCAT('%', #{keyword}, '%')
    AND deleted = 0
</select>
```

**IN 查询**：

```xml
<!-- MUST: 使用 foreach -->
<select id="findByIds" resultType="Product">
    SELECT id, name, price FROM product
    WHERE id IN
    <foreach collection="ids" item="id" open="(" separator="," close=")">
        #{id}
    </foreach>
</select>
```

### 5.3 审计要求

Checkstyle 已配置禁止 `` `${}` `` 的检查，CI 自动阻断。

季度安全扫描使用 SQLMap 对所有 API 进行注入测试。

## 6. XSS 防护

> MUST：所有用户输入必须经过净化处理，防止 XSS 攻击。

### 6.1 CSP Header 配置

MUST：所有页面响应包含 Content-Security-Policy Header。

```text
Content-Security-Policy:
  default-src 'self';
  script-src 'self' 'nonce-{{random}}' https://cdn.example.com;
  style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
  img-src 'self' data: https://cdn.example.com https://*.qiniu.com;
  font-src 'self' https://fonts.gstatic.com;
  connect-src 'self' https://api.example.com;
  frame-ancestors 'none';
  base-uri 'self';
  form-action 'self' https://api.example.com;
```

### 6.2 输出编码规则

| 输出位置 | 编码方式 | 说明 |
|----------|----------|------|
| HTML 内容 | HTML Entity 编码 | `<` → `&lt;` |
| HTML 属性 | 属性值编码 | `'` → `&#x27;` |
| JavaScript | JS 编码 | Unicode 转义 |
| URL | URL 编码 | `encodeURIComponent()` |
| CSS | CSS 编码 | 转义非字母数字字符 |
| JSON API | JSON 序列化 | 使用 Jackson 默认配置 |

### 6.3 v-html 安全规则

MUST：Vue 中使用 `v-html` 时必须净化 HTML 内容。

```vue
<!-- MUST: 使用 DOMPurify 净化 -->
<template>
  <div v-html="sanitizedHtml"></div>
</template>

<script setup>
import DOMPurify from 'dompurify'

const props = defineProps({ content: String })
const sanitizedHtml = computed(() => {
  if (!props.content) return ''
  return DOMPurify.sanitize(props.content, {
    ALLOWED_TAGS: ['b', 'i', 'u', 'p', 'br', 'span', 'a', 'img', 'ul', 'ol', 'li'],
    ALLOWED_ATTR: ['href', 'target', 'class', 'src', 'alt']
  })
})
</script>
```

MUST：禁止在 `v-html` 中渲染未净化的用户输入。

### 6.4 HttpOnly Cookie

MUST：Session Token / JWT 设置 HttpOnly + Secure + SameSite。

```text
Set-Cookie: SESSION_ID=xxx; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=3600
```
