# 后端规范快速参考卡 | Backend Quick Reference

> 完整规范见《[后端规范](后端规范.md)》，本文档仅列出MUST级别核心规则。

## 核心架构规则

| 规则 | 要求 | 规范章节 |
|------|------|----------|
| 分层架构 | Controller→Service→Mapper，禁止跨层调用 | 1.2 |
| Controller职责 | 仅参数校验+权限校验+调用Service，禁止业务逻辑 | 2.1 |
| Service接口 | 所有Service必须定义接口+实现类 | 2.2 |
| 依赖注入 | 构造器注入（`@RequiredArgsConstructor` + final），禁止`@Autowired`字段注入 | 5.1 |
| 统一响应 | 所有接口返回`Result<T>`包装结构，含code/message/data/traceId | 2.1 |

## API设计规则

| 规则 | 要求 |
|------|------|
| URL命名 | 小写+连字符+复数名词，如 `/api/users/{id}` |
| HTTP方法 | GET查询/POST创建/PUT全量更新/PATCH部分更新/DELETE删除 |
| 状态码 | 200成功/201创建/400参数错误/401未认证/404不存在/500服务器错误 |
| 分页参数 | `?page=1&size=20&sort=createdAt:desc` |

## 数据库规则

| 规则 | 要求 |
|------|------|
| 禁止SELECT * | 必须明确列出所需字段 |
| 表名命名 | 小写+下划线，`业务_表用途` |
| 索引命名 | pk_/uk_/idx_ 前缀 |
| 分页限制 | 最大分页 <= 1000，深分页用游标方案 |

## 异常处理规则

| 规则 | 要求 |
|------|------|
| 错误码 | 必须使用枚举类（ErrorCodeEnum），禁止魔法数字 |
| 业务异常 | 抛出BusinessException，保留原始异常链 |
| 全局处理 | @RestControllerAdvice统一处理 |
| 日志级别 | 业务异常WARN/数据异常ERROR/未知异常ERROR+完整堆栈 |

## 关键配置速查

```yaml
# MyBatis-Plus
mybatis-plus:
  mapper-locations: classpath:mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true

# HikariCP（通用服务）
spring.datasource.hikari:
  minimum-idle: 5
  maximum-pool-size: 20  # 高并发服务: 50

# Micrometer Tracing
management:
  tracing:
    sampling:
      probability: 1.0  # 生产: 0.1
```
